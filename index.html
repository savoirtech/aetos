Aetos - Integration at its finest

Aetos is the microservices integration container built on a solid Java-based Apache platform. Its made up of Apache Karaf, Camel, CXF, and ActiveMQ with the latest and most stable versions. But isn't this Apache Servicemix? Not at all! We created Aetos to provide a lightwieght container without a lot of extra overhead that includes the proverbial kitchen sink. We offer several versions of Aetos based on the different major versions of Karaf as a download from Maven Central.


The Aetos code base is a magic maven build that allows you to produce your own integration container with the versions that you need, that are based on the pure Apache implementations of Camel, Karaf, CXF, ActiveMQ, and many others. No more need to rely on companies who fork Apache code to produce their own stacks and force you to wait for them to give you a patch on thier own time frames. That is what is called vendor lock in. With Aetos, you have a stack that is based on pure Apache software (like clean water), ASLv2 Licensed, and the versions that are right for your organization. Want the latest and greatest all built for you? They are ready for you to download at Maven Central!


What makes Aetos different is you can "roll your own" microservices container to fit your needs.  No more waiting for a project or vendor to update a buggy component.  You can do it yourself!  Just checkout or fork the code base and update your versions in the maven POM, build it, and you are in business.  Your microservice container, just the way you want it.  Simple as pie! 

Microservices - The New Application Development Frontier

Microservices is the new way to deploy an application.  Instead of deploying a full web stack in one location, the application is distributed.  For example, you may have some javascript and web assets like a single HTML page deployed from a web server or container.  You can even use Aetos as a web endpoint to deliver these assets and not be tied to a web container or other server.  That client side javascript is then executed at the client and the back end hits many distributed services that make up the back end on different servers/docker instances/cloud locations.  The servers can even work together by imlementing ActiveMQ or other persistent messaging to guarantee that no message is lost.  It offers instant clustering in the face of this new distributed architecture.

WHAT'S IN IT?

Aetos is an Apache Karaf container bult with some of the best Apache projects embedded inside.   These projects are best-in-breed, meaning they are the top quality projects that are at times included as part of other commercial containers.  Why Apache Karaf?  Because it's a full service modular container (more on that later) with an OSGi runtime as a kernel subsystem.  Its kind of like running your own Java based operating system, with a full SSH console to boot!  The components are the API's that let you build your services and routes which you can expose to the outside world.  What are these awesome Apache projects?

Apache Camel

Best service router money can buy! (Free)

Apache Camel is the mostly wildly popular routing framework around.  Offering more than 210 different typs of components and endpoints, no other API offers more capabilities to connect to the outside world or external systems.

Apache ActiveMQ

Persistant Messaging

Apache ActiveMQ is the backbone for ensuring your services get to their destination without the need for point-2-point issues.  Never lose a message due to a server that is down and provide instant clustering for a competing consumer to your services.

Apache CXF

The premier RESTful and Web Service API

Apache CXF is THE most popular and well known RESTful and Web Service container and API.  So well known that it has become the staple API within many popular JavaEE containers like JBoss and Websphere.  Apache CXF supports the Java JAX-RS and JAX-WS standards.

Apache Karaf

Container that brings modularity to it all

Apache Karaf is the container that provides lifecycle and modularity to your services.  Start, stop, deploy, and undeploy through an SSH console without any impacts on the classloading, or obtaining the dreaded OutOfMemoryException.

MODULARITY

The ultimate goal of a true microservices container

"Do you really need that fat stack? Most deployments use 5-10% of a Java EE stack, so why eat those resources?"

What makes Aetos different than all the rest?  Modularity.  Modularity is key to any Microservice container so you can deploy only those services and technologies that are right for your applications.  Modularity is cloud friendly, so you can deploy and undeploy your services without an impact on the conatiner.  Java is notorious for its tree-based classloader and today's Java EE containers are typically monolithic, where by you Get It Allâ„¢.  Cloud images can be challenging to run full-stack containers due to memory requirements.  Do you really need that fat stack?  Most deployments use 5-10% of a Java EE stack, so why eat those resources?


Aetos is OSGi-based which means it has a matrix-based classloader.  You can load and unload your services at will, or even hot-upgrade them without having to restart your container!


Do you want a full profile that a Java EE stack has?  Throw together a feature for the  profile that you require, and you will have the same things that a Java EE container will offer.  JTA, JAX-RS, JAX-WS, you name it.  Need security?  JAAS is baked right in.  Don't need the full profile?  "Feature" up what only you need and keep that container as light as you want.


What about microprofile?  Its all ready there!  Pull in the bundles for microprofile and you are microprofile ready.


Shhhh... we don't like profiles any ways.  We think that you should put in your container what you want and need, not what some group of people think that you should have. :-)

Console Made with Apache Karaf

Apache Karaf is the cool lightweight container that makes this a great engine to manage your services.  Apache Karaf is an OSGi-based runtime which provides a lightweight container onto which various components and applications can be deployed.

Apache Karaf uses either Apache Felix Framework or Eclipse Equinox OSGi frameworks, and provides additional features on top of the framework.  Apache Karaf can be scaled from a very lightweight container to a fully featured enterprise service: it's a very flexible and extensible container, covering all the major needs.

What makes this great is it can be managed through a terminal since it offers a really awesome console.  Devops folks love it because they can manage and monitor Karaf with Puppet, Apache Chef, or just about any other orchestration software for cloud based deployments.
